## How It Works

The key API is `layoutNextLine` from `react-pretext` — an iterator that lays out one line at a time, accepting a different `maxWidth` on each call.

```ts
import { computePreparedText, layoutNextLine } from "react-pretext";

const prepared = computePreparedText({ text, font });
let cursor = { segmentIndex: 0, graphemeIndex: 0 };
let y = 0;

while (true) {
  const width = getAvailableWidth(y, obstacle); // varies per line
  const line = layoutNextLine(prepared, cursor, width);
  if (line === null) break;

  ctx.fillText(line.text, xOffset, y);
  cursor = line.end;
  y += lineHeight;
}
```

For lines that intersect the obstacle, the demo lays out text into the left gap first, then continues remaining text into the right gap on the same y-coordinate — so text flows on both sides of the shape.
