These functions can be used outside of React — in Node.js scripts, build tools, or any JavaScript context.

## computeLayout

Full layout computation that returns line-level details. Results are cached in a global LRU cache (500 entries).

```ts
import { computeLayout } from "react-pretext";

const result = computeLayout({
  text: "Hello world, this is a long paragraph that will wrap.",
  font: "16px Inter",
  maxWidth: 300,
  lineHeight: 24,
});

console.log(result.height);    // total height in px
console.log(result.lineCount); // number of lines
console.log(result.lines);     // array of PretextLineInfo
```

### Options

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `text` | `string` | Yes | Text content to lay out |
| `font` | `string` | Yes | CSS font shorthand (e.g. `"16px Inter"`) |
| `maxWidth` | `number` | Yes | Maximum width in pixels for line breaking |
| `lineHeight` | `number` | Yes | Line height in pixels |
| `locale` | `string` | No | Locale for text segmentation |

### Return Type

Returns a [`PretextLayoutResult`](/docs/api/types#pretextlayoutresult).

<Callout type="info">
  On the server (no `document`), `computeLayout` returns an estimated layout using a character-width heuristic (8px average). See [SSR](/docs/guides/ssr) for details.
</Callout>

## computeHeight

Fast path that returns only the total height. Uses `prepare()` + `layout()` instead of `prepareWithSegments()` + `layoutWithLines()` — skips line text materialization. ~10x faster than `computeLayout`. Cached separately with a 20k entry cache.

```ts
import { computeHeight } from "react-pretext";

const height = computeHeight({
  text: "Hello world",
  font: "16px Inter",
  maxWidth: 300,
  lineHeight: 24,
});
```

Takes the same options as `computeLayout`. Returns a single `number` (height in pixels).

<Callout type="info">
  Use `computeHeight` when you only need dimensions — for example, in a virtualizer's height computation. The virtualizer uses this internally.
</Callout>

## createCache

Creates a standalone LRU cache instance. Use this if you want isolated caching separate from the global cache.

```ts
import { createCache, computeLayout } from "react-pretext";

const cache = createCache(1000); // capacity of 1000 entries

const result = computeLayout(
  { text: "Hello", font: "16px Inter", maxWidth: 300, lineHeight: 24 },
  cache,
);
```

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `capacity` | `number` | No | Maximum number of entries before LRU eviction (default: 500) |

**Returns:** A `LayoutCache` object with `get`, `set`, `clear`, and `size`.

---

## clearLayoutCache

Clears the global layout cache. Call this when fonts change or when you need to free memory.

```ts
import { clearLayoutCache } from "react-pretext";

clearLayoutCache();
```

<Callout type="warn">
  This only clears the global layout cache (used by `computeLayout` and `usePretextLayout`). The height cache used by `computeHeight` is separate. Use `clearAllCaches` to clear everything.
</Callout>

---

## clearAllCaches

Clears all caching layers: the react-pretext layout cache, height cache, and the underlying `@chenglou/pretext` measurement caches.

```ts
import { clearAllCaches } from "react-pretext";

clearAllCaches();
```

## computePreparedText

Prepares text for layout with SSR fallback and locale handling. Use the returned value with `layoutNextLine` for per-line variable widths.

```ts
import { computePreparedText, layoutNextLine } from "react-pretext";

const prepared = computePreparedText({ text: "Hello world", font: "16px Inter" });
let cursor = { segmentIndex: 0, graphemeIndex: 0 };

const line = layoutNextLine(prepared, cursor, 300);
```

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `text` | `string` | Yes | Text content to prepare |
| `font` | `string` | Yes | CSS font shorthand |
| `locale` | `string` | No | Locale for text segmentation |
| `whiteSpace` | `"normal" \| "pre-wrap"` | No | Whitespace handling mode |

**Returns:** A `PreparedTextWithSegments` for use with `layoutNextLine`.

## computeVariableLayout

Lays out text with a different `maxWidth` per line. Handles SSR fallback, locale, and cursor management internally.

```ts
import { computeVariableLayout } from "react-pretext";

const result = computeVariableLayout({
  text: "Long text that wraps around an obstacle...",
  font: "16px Inter",
  lineHeight: 24,
  getMaxWidth: (lineIndex, y) => {
    // Return available width for this line based on obstacle geometry
    return isObstacleAtY(y) ? 200 : 400;
  },
});
```

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `text` | `string` | Yes | Text content to lay out |
| `font` | `string` | Yes | CSS font shorthand |
| `lineHeight` | `number` | Yes | Line height in pixels |
| `getMaxWidth` | `(lineIndex, y) => number` | Yes | Returns available width for each line |
| `locale` | `string` | No | Locale for text segmentation |
| `whiteSpace` | `"normal" \| "pre-wrap"` | No | Whitespace handling mode |
| `maxLines` | `number` | No | Safety limit on iterations (default: 1000) |

**Returns:** A `VariableLayoutResult` with `height`, `lineCount`, and `lines` (each line includes its `maxWidth`).

## computeLineRanges

Iterates over line ranges without materializing line text. Useful for aggregate geometry calculations.

```ts
import { computeLineRanges } from "react-pretext";

const lineCount = computeLineRanges({
  text: "Hello world",
  font: "16px Inter",
  maxWidth: 300,
  onLine: (range) => {
    console.log(range.width, range.start, range.end);
  },
});
```

**Returns:** The total line count.

---

## profileLayout

Returns performance profiling data for text preparation. Useful for debugging and benchmarking.

```ts
import { profileLayout } from "react-pretext";

const profile = profileLayout({ text: "Hello world", font: "16px Inter" });
console.log(profile.totalMs, profile.analysisSegments);
```

**Returns:** A `PrepareProfile` with `analysisMs`, `measureMs`, `totalMs`, `analysisSegments`, `preparedSegments`, `breakableSegments`.

---

## setLocale

Sets the locale for text analysis (word/grapheme segmentation). Also clears the underlying measurement caches.

```ts
import { setLocale } from "react-pretext";

setLocale("ja-JP");
```
